# Bulk Console

แดชบอร์ดคำนวณแคลอรี่ BMR/TDEE และอัตราการเพิ่มน้ำหนักต่อวัน พร้อมบันทึกเนื้อสัตว์/เวย์โปรตีน/คาร์บที่กินจริง แยกตามมื้อ (เช้า/กลางวัน/เย็น) แล้วเทียบกับเป้าหมายแบบเรียลไทม์ พร้อมหน้าประวัติย้อนหลังดูข้อมูลโภชนาการแต่ละวัน

เป็น static site ล้วน (HTML/CSS/JS) ไม่มี backend ไม่มี build step — เปิดไฟล์ `index.html` ก็ใช้งานได้ทันที หรือจะ deploy ขึ้น GitHub Pages ก็ได้ฟรี

## โครงสร้างไฟล์

```
calorie-dashboard/
├── index.html          หน้าหลัก (บันทึกอาหารรายมื้อ + สรุปวันนี้)
├── history.html         หน้าประวัติย้อนหลัง (ดูข้อมูลโภชนาการแต่ละวัน)
├── assets/
│   ├── style.css        ธีม (dark console)
│   ├── data.js          ฐานข้อมูลโภชนาการ (เนื้อสัตว์ / คาร์บ / คำแปลไทย-อังกฤษ)
│   ├── script.js        ตรรกะการคำนวณของหน้าหลัก
│   └── history.js        ตรรกะของหน้าประวัติย้อนหลัง
└── README.md
```

## บันทึกอาหารแยกตามมื้อ

หน้าหลักแบ่งการบันทึกอาหารเป็น 3 แท็บ: **เช้า / กลางวัน / เย็น** แต่ละมื้อมีตารางเนื้อสัตว์และคาร์บของตัวเอง พร้อมสรุปยอดต่อมื้อ ส่วนเวย์โปรตีนนับเป็นยอดรวมทั้งวัน (ไม่ผูกกับมื้อใดมื้อหนึ่ง) หน้าสรุป (05) จะรวมทุกมื้อเป็นยอดวันนี้ พร้อมตารางแยกยอดรายมื้ออีกที

แต่ละแถวอาหารต้องกด **"✓ บันทึก"** ก่อนถึงจะถูกนับรวมในยอดแคลอรี่/โปรตีน — ระหว่างที่ยังไม่กดบันทึก ตัวเลขโปรตีน/ไขมัน/คาร์บที่โชว์เป็นแค่ตัวอย่าง (สีจาง) ยังไม่ถูกนับ ป้องกันข้อมูลที่พิมพ์ค้างหรือยังไม่ครบถูกนับรวมโดยไม่ตั้งใจ กดปุ่มซ้ำ ("แก้ไข") เพื่อแก้ไขแถวที่บันทึกไปแล้วได้ทุกเมื่อ

## ประวัติย้อนหลัง

ทุกครั้งที่กรอกข้อมูล ระบบจะบันทึกยอดของ "วันนี้" (ตามวันที่เครื่องของคุณ) ไว้ใน `localStorage` ของเบราว์เซอร์อัตโนมัติ กดปุ่ม **"📅 ประวัติย้อนหลัง"** ที่มุมขวาบนของหน้าหลัก จะเปิดหน้าต่างใหม่ (`history.html`) แสดงรายวันย้อนหลังทั้งหมด พร้อมยอดรวมเทียบเป้าหมายและยอดแยกตามมื้อ ลบเป็นวันหรือลบทั้งหมดได้จากหน้านั้น

ข้อมูลประวัติอยู่ใน localStorage ของเบราว์เซอร์เครื่องนั้นเท่านั้น (ไม่ sync ข้ามอุปกรณ์ ไม่ขึ้น server ใดๆ) ถ้าล้าง browser data หรือเปลี่ยนเครื่อง ประวัติจะหายไป

## รันในเครื่องตัวเอง

ไม่ต้องติดตั้งอะไรเลย เปิด `index.html` ด้วยเบราว์เซอร์ได้โดยตรง หรือถ้าอยากรันผ่าน local server (กัน CORS/cache issue บางเบราว์เซอร์):

```bash
cd calorie-dashboard
python3 -m http.server 8000
# แล้วเปิด http://localhost:8000
```

## ตั้งค่าขึ้น GitHub (ครั้งแรก)

1. สร้าง repo ใหม่บน GitHub (ปุ่ม **New repository**) ตั้งชื่อ เช่น `bulk-console` — ไม่ต้องติ๊ก "Add README" เพราะมีอยู่แล้ว

2. ในเครื่อง เข้าไปที่โฟลเดอร์โปรเจกต์แล้วรัน:

```bash
cd calorie-dashboard
git init
git add .
git commit -m "Initial commit: Bulk Console dashboard"
git branch -M main
git remote add origin https://github.com/<username>/bulk-console.git
git push -u origin main
```

แทน `<username>` ด้วยชื่อบัญชี GitHub ของคุณ

## เปิดใช้งานผ่าน GitHub Pages (ฟรี ไม่ต้องมี server)

1. ไปที่ repo บน GitHub → **Settings** → **Pages** (เมนูซ้าย)
2. ใต้ **Build and deployment** เลือก Source = **Deploy from a branch**
3. เลือก Branch = `main`, Folder = `/ (root)` แล้วกด **Save**
4. รอประมาณ 1-2 นาที จะได้ลิงก์ประมาณ:
   `https://<username>.github.io/bulk-console/`

เข้าใช้งานได้จากมือถือ/คอมทุกเครื่องผ่านลิงก์นี้ ไม่ต้อง clone ซ้ำ

## อัปเดตหลังจากแก้ไขไฟล์

```bash
git add .
git commit -m "อัปเดต: <อธิบายสิ่งที่แก้>"
git push
```

GitHub Pages จะ deploy เวอร์ชันใหม่ให้อัตโนมัติภายในไม่กี่นาที

## ค้นหาอาหารที่ไม่มีในลิสต์ (ออนไลน์)

นอกจากรายการที่ตั้งไว้ล่วงหน้าใน `assets/data.js` แล้ว ยังพิมพ์ค้นหาอาหารอื่น ๆ แบบเรียลไทม์ได้ (เช่น หมูสับ, ปลาหมึก, ไข่ต้ม) ผ่าน [USDA FoodData Central](https://fdc.nal.usda.gov/) ซึ่งเป็นฐานข้อมูลโภชนาการฟรี:

1. ขอ API Key ฟรีที่ https://fdc.nal.usda.gov/api-key-signup.html (กรอกอีเมล รับคีย์ทันที ไม่มีค่าใช้จ่าย)
2. วางคีย์ในช่อง "USDA API Key" บนหน้าเว็บ — คีย์จะถูกเก็บไว้ใน `localStorage` ของเบราว์เซอร์เครื่องนั้น ๆ เท่านั้น ไม่ถูกส่งขึ้น repo หรือ server ใด ๆ
3. พิมพ์ชื่ออาหาร (ไทยหรืออังกฤษ) แล้วกด "+ เพิ่ม" ที่ผลลัพธ์ที่ต้องการ ระบบจะดึงเข้าตารางอาหารพร้อมให้กรอกน้ำหนักเป็นกรัมได้เลย

หมายเหตุ: USDA เป็นฐานข้อมูลภาษาอังกฤษ เว็บนี้แปลคำไทยที่พบบ่อย ๆ ให้อัตโนมัติ (ดูรายการแปลได้ใน `THAI_FOOD_TERMS` ที่ `assets/data.js`) ถ้าพิมพ์แล้วไม่เจอผลลัพธ์ ลองพิมพ์เป็นภาษาอังกฤษแทน (เช่น "pork ground raw")

## ปรับแต่งเพิ่มเติม

- เพิ่ม/แก้รายการเนื้อสัตว์หรือคาร์บ → แก้ที่ `assets/data.js` (ค่าเป็นต่อ 100 กรัม)
- เพิ่ม/แก้คำแปลไทย→อังกฤษสำหรับค้นหาออนไลน์ → แก้ที่ `THAI_FOOD_TERMS` ใน `assets/data.js`
- ปรับสูตรคำนวณ (BMR, surplus, สัดส่วนโปรตีน) → แก้ที่ `assets/script.js` ฟังก์ชัน `calc()`
- ปรับสีธีม/ฟอนต์ → แก้ตัวแปรใน `:root` ที่หัวไฟล์ `assets/style.css`

## หมายเหตุ

ค่าพลังงานที่ใช้: 1 กก. น้ำหนักตัว ≈ 7,700 kcal, BMR คำนวณด้วยสูตร Mifflin-St Jeor ค่าทั้งหมดเป็นค่าประมาณเพื่อการวางแผนทั่วไป ไม่ใช่คำแนะนำทางการแพทย์ — หากมีโรคประจำตัวหรือข้อกังวลด้านสุขภาพ ควรปรึกษาแพทย์หรือนักโภชนาการ
